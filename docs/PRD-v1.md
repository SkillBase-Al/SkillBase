# SkillBase — AI 技能商店 产品需求文档 v1.0

> 一个面向普通用户的桌面 GUI 应用，让 AI 技能的发现、安装、管理和共享像手机 App Store 一样简单。

---

## 1. 产品背景与定位

### 1.1 市场背景

2025 年底，Anthropic 发布了 Agent Skills 开放标准（`SKILL.md`，[agentskills.io](https://agentskills.io)），定义了 AI Agent 技能的通用格式。截至 2026 年初：
- 85,000+ 公开 Skill，30+ AI Agent 平台已支持（Claude Code、OpenAI Codex CLI、Cursor、Windsurf、GitHub Copilot 等）
- 多个开源注册表/市场出现：SkillNet（20 万+ Skill，5 维评估）、SkillsMP（28 万+）、ClawHub 等
- SkillNet 的研究表明，使用高质量 Skill 可使 Agent 平均奖励提升 ~40%，执行步骤减少 ~30%

然而，现有工具存在明显缺口：
- **普通用户缺乏 GUI 工具** — CLI 界面（MagicSkills）对非技术用户门槛高
- **缺乏统一发现入口** — Skill 散落在 GitHub、不同市场、社区帖子中，无聚合平台
- **缺乏可视化质量管理** — 无法直观了解 Skill 质量、安全风险
- **缺乏去重管理** — 用户安装大量 Skill 后冗余堆积，无检测工具

### 1.2 产品定位

**SkillBase — AI 技能的 App Store**

一个桌面 GUI 应用，让普通用户像在手机应用商店一样：
- **发现**：浏览、搜索、排行榜，找到需要的 AI 技能
- **安装**：一键安装到本地的多个 AI Agent
- **管理**：查看已安装 Skill，标记启/禁用，分组管理
- **评估**：格式化合规检查、安全检查、质量评分
- **保洁**：自动检测名称/描述相似的冗余 Skill，帮助清理

### 1.3 目标用户

- **AI 工具普通用户**（核心）：使用 Claude Code、Cursor、Windsurf 等的开发者/创作者，希望图形化管理 AI 技能，不懂也不想用 CLI
- **Skill 贡献者**：在 GitHub 等平台创作和开源 Skill 的作者，通过此平台获得曝光和用户反馈
- **进阶用户**：需要批量管理、质量审计、团队同步的高级用户

## 2. 功能需求

### 2.1 Skill 商店（内置市场）

#### 2.1.1 内容聚合引擎
通过各平台的官方公开 API 获取 SKILL.md 格式的 Skill 元数据：

| 数据源 | 类型 | 预期规模 |
|--------|------|----------|
| GitHub (公开仓库) | 通过 GitHub API 搜索 SKILL.md 文件 | 持续增长 |
| SkillNet (zjunlp) | 语义搜索 + 5 维评分 API | 20 万+ |
| SkillsMP | 聚合市场 API | 28 万+ |
| Anthropic 官方 Skill 仓库 | GitHub [anthropics/skills](https://github.com/anthropics/skills) | 数十 |

爬取的 Skill 数据在服务端索引后，客户端通过 API 获取目录。

#### 2.1.2 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **首页推荐** | 精选 Skill、热门排行、新上架分类 | P0 |
| **分类浏览** | 按领域分类（开发、写作、数据分析、设计、运维等） | P0 |
| **搜索** | 关键词搜索 + 语义搜索（基于向量索引） | P0 |
| **Skill 详情页** | 名称、描述、评分、截图/示例、安装量、依赖信息、安全状态 | P0 |
| **用户评价系统** | 评分和文字评价（需要用户登录） | P1 |
| **排行榜** | 按安装量、评分、更新时间排序 | P1 |

#### 2.1.3 Skill 卡片字段

Skill 从市场获取时展示以下信息：

```
- 名称 (name)
- 描述 (description)
- 作者/来源 (source)
- 综合评分 (0-5，综合各评估维度)
- 安全等级 (Safe / Warning / Dangerous)
- SKILL.md 合规得分 (0-100)
- 安装量统计
- 标签/分类
- GitHub Star 数 (如有)
- 最后更新时间
```

### 2.2 本地 Skill 管理

#### 2.2.1 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **已安装 Skill 列表** | 展示所有已安装到本地的 Skill | P0 |
| **一键安装** | 从市场选择 Skill → 选择一个或多个目标 Agent → 自动安装 | P0 |
| **更新到 Agent** | 当 Skill 有更新（本地修改或市场新版本），一键覆盖推送到已安装的目标 Agent 目录 | P0 |
| **多 Agent 同步** | 一次操作将同一 Skill 同时安装/更新到多个 Agent 目录 | P0 |
| **卸载** | 从本地移除指定 Skill（同时从所有 Agent 目录删除） | P0 |
| **启用/禁用** | 临时禁用某 Skill 而不删除 | P0 |
| **批量操作** | 批量安装、卸载、更新、启用/禁用 | P1 |
| **导入/导出** | 导出 Skill 集合为 ZIP 或 Git 仓库，支持导入他人分享的集合 | P1 |

#### 2.2.2 Agent 集成

支持用户配置本地有哪些 AI Agent 工具，以及它们的 Skill 目录路径：

| Agent | Skill 目录位置 |
|-------|---------------|
| Claude Code | `CLAUDE.md` / `~/.claude/skills/` |
| Cursor | `.cursor/rules/` |
| Windsurf | `.windsurfrules/` |
| Codex CLI | `.codex/skills/` |
| GitHub Copilot | `.github/copilot-instructions.md` |
| 支持 SKILL.md 标准的其他 Agent | 用户自定义路径 |

#### 2.2.3 Skillset（技能集合）

用户可以将 Skill 组织到不同的 **Skillset（集合）** 中，每个 Skillset 对应不同的项目或工作场景：

- 创建/删除/重命名 Skillset
- 从已安装 Skill 中添加/移除
- 将 Skillset 同步到指定项目的 Agent 配置

### 2.3 Skill 质量评估

#### 2.3.1 评估维度

| 维度 | 检查内容 | 检测方式 |
|------|----------|----------|
| **格式合规** | SKILL.md 是否符合 agentskills.io 规范（YAML 字段完整性、命名规范、描述长度等） | 本地解析校验 |
| **安全检查** | 检测脚本中是否含潜在危险命令（rm -rf、curl 恶意 URL 等） | 静态分析 |
| **依赖完整性** | 检测 SKILL.md 中提到的依赖（Python 包、系统工具等）是否可解析 | 扫描 `scripts/` + 文档分析 |
| **质量评分** | 描述清晰度、指令完整性、示例丰富度等 | 服务端 LLM 评估（每月免费额度，超额可绑定个人 API Key） |

#### 2.3.2 评估结果展示

- 每个本地 Skill 显示综合质量仪表盘
- 格式合规/安全/质量各维度打分（0-100 分）
- 问题列表：具体不符合项及修复建议
- 批量扫描所有已安装 Skill 的功能

### 2.4 去重检测

#### 2.4.1 检测范围

| 检测类型 | 方法 | 精度 |
|----------|------|------|
| **名称相似** | 模糊字符串匹配（Levenshtein、Jaccard 等） | 高 |
| **描述相似** | 文本向量化 + 余弦相似度 | 中 |
| **完全重复** | SKILL.md content hash 匹配 | 精确 |

#### 2.4.2 功能流程

1. 用户点击"检查重复"
2. 系统扫描本地所有已安装 Skill
3. 展示相似 Skill 对列表，标注相似度和重叠区域
4. 用户可查看对比（并排显示两个 SKILL.md）
5. 用户自行决定保留/删除哪些

> **注**：v1 不做自动合并，只检测和标记，由用户手动决策。

### 2.5 设置与配置

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **Agent 路径配置** | 添加/编辑本地的 AI Agent 安装路径 | P0 |
| **Skill 存储目录** | 自定义全局 Skill 仓库路径 | P0 |
| **数据源配置** | 启用/禁用特定数据源 | P1 |
| **自动更新检查** | 定期检查已安装 Skill 是否有新版本 | P1 |
| **代理配置** | 网络代理设置 | P1 |

### 3.2 技术选型

| 层 | 技术 | 理由 |
|----|------|------|

| **前端** | React 18 + TypeScript | 生态成熟，组件库丰富 |
| **UI 框架** | Tailwind CSS + shadcn/ui | 快速构建美观界面 |
| **状态管理** | Zustand | 轻量、高性能 |
| **Rust 后端** | - reqwest（HTTP 客户端） | 爬虫/API 请求 |
| | - serde（序列化） | 数据解析 |

| | - syntect（语法高亮） | Skill 代码展示 |
| | - ignore（Gitignore 风格忽略规则） | 文件扫描 |

### 3.4 服务端爬虫架构

爬虫引擎运行在独立后端服务上，定时从各数据源通过**官方公开 API** 拉取数据，建立统一索引。所有操作均在合规范围内进行。

**合规准则：**
- 仅通过各平台的**官方公开 API** 获取数据（GitHub REST API、SkillNet API），不爬取网页、不绕过认证
- 遵守各 API 的速率限制和调用频率约束
- 仅索引 MIT / Apache-2.0 许可证的 SKILL.md（通过 API 返回的 license 字段判断）
- 服务端存储完整 SKILL.md 内容副本用于客户端安装（MIT/Apache-2.0 许可允许分发）
- 始终在 UI 中标注原始作者、仓库链接和许可证信息
- 遵循 GitHub API 的 `-H "Accept: application/vnd.github+json"` 规范使用

### 3.5 去重检测实现

**第一阶段（v1，本版本实现）：**

1. **名称相似度**：对 `name` 字段做标准化处理后计算 Levenshtein 距离 + Jaccard 相似度
2. **描述相似度**：将描述文本发送至服务端 API，由服务端做向量化 + 余弦相似度计算

### 3.6 服务端部署方案

| 阶段 | 方案 | 说明 |
|------|------|------|
| **初期 (v1.0)** | 云服务器 (VPS) | 部署爬虫调度器 + REST API + PostgreSQL + 向量嵌入服务，最低配置 2C4G |
| **后期** | Serverless 迁移 | 待业务量明确后，可将 REST API 和向量计算迁移至 Serverless 架构以降低成本 |

**v1.0 服务器最低配置需求：**

| 资源 | 规格 |
|------|------|
| CPU | 2 核 |
| 内存 | 4 GB |
| 存储 | 50 GB SSD |
| 带宽 | 基础的出站带宽即可 |
3. **阈值设定**：相似度 > 0.8 标记为"高度相似"，> 0.6 标记为"可能相似"
4. **结果展示**：在 UI 中以卡片对形式展示相似 Skill，并排显示差异

## 5. 数据流

### 5.1 发现 → 安装流程

```
用户浏览市场 → 点击 Skill 进入详情 → 查看评分/安全评估/评价
    → 点击"安装" → 选择一个或多个目标 Agent（多选：Claude Code / Cursor / Windsurf 等）
    → Rust 后端下载 SKILL.md → 校验格式 → 安全检查
    → 逐一写入每个目标 Agent 的 Skill 目录
    → 更新本地索引 + 记录安装映射关系（Skill → Agent 列表）
    → UI 刷新 → 安装成功提示
```

### 5.2 更新覆盖流程

```
场景 A: 用户本地修改 SKILL.md 后
    → 检测到变更 → 提示"检测到 Skill 变更，是否更新到 Agent？"
    → 用户确认 → Rust 后端查找该 Skill 已安装到的所有 Agent
    → 逐一覆盖写入每个 Agent 目录 → 完成提示

场景 B: 市场中的 Skill 发布新版本
    → 服务端检测到更新 → 客户端提示"xxx Skill 有新版本"
    → 用户点击更新 → 下载最新 SKILL.md → 校验格式 → 安全检查
    → 覆盖已安装的所有 Agent 目录 → 完成提示
```

### 5.3 去重检测流程

```
用户点击"扫描重复"
    → Rust 后端遍历 ~/.skillbase/skills/ 下所有 SKILL.md
    → 提取 name + description
    → 本地计算名称 Levenshtein 距离 + Jaccard 相似度
    → 将描述文本批量发送至服务端 API 做向量化 + 余弦相似度计算
    → 按阈值分组（>0.8 高度相似，>0.6 可能相似）
    → 结果返回前端渲染
```

### 5.4 服务端数据采集流程

```
定时器触发 (每 6 小时)
    → GitHub 适配器: 搜索 "filename:SKILL.md" 的仓库 → 获取原始内容
    → SkillNet 适配器: 调用其搜索 API → 获取元数据
    → SkillsMP 适配器: 调用其列表 API → 获取元数据
    → 统一管道处理:
        1. hash 去重 (同一内容不重复索引)
        2. 许可证过滤 (仅 MIT / Apache-2.0)
        3. SKILL.md 格式校验
        4. 基础安全检查
        5. 写入 PostgreSQL 索引
    → REST API 对外提供搜索/浏览/详情接口
    → 客户端通过 API 消费数据
```

## 6. 非功能性需求

### 6.1 性能

| 指标 | 目标 |
|------|------|
| 应用启动时间 | < 3 秒（冷启动）|
| 市场浏览响应 | < 500ms（翻页/搜索） |
| 本地列表加载 | < 1 秒（1000 个 Skill） |
| 去重扫描 | < 5 秒（500 个 Skill） |
| 安装单个 Skill | < 3 秒 |
| 内存占用 | < 200MB（空闲）|
| 磁盘占用（应用） | < 100MB |

### 6.2 安全

- 所有爬取内容经过基础安全检查后再展示
- 安装到本地的 Skill 经过安全扫描标记危险命令
- 不自动执行任何从市场获取的脚本
- 不允许 Skill 修改应用自身配置
- 网络请求可配置代理

### 6.3 跨平台

| 平台 | 最低支持版本 | 安装包格式 |
|------|------------|-----------|
| macOS | 13.0+ | .dmg / .app |
| Windows | 10+ | .msi / .exe |
| Linux | 主流发行版 | .AppImage / .deb |
